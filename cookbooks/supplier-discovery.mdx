---
title: "Supplier discovery"
description: "Given a product and a location, return a clean supplier shortlist, manufacturers and distributors, every time."
---

**Inputs:** `product` (free text) · `location` (a country). **Returns:** a deduped shortlist of suppliers.

A bare `search_organizations(query="macbook dubai")` mixes in repair shops, rental firms, and parts makers, and the order drifts between runs. The quality comes from disciplined parameters, this template bakes them in.

## The recipe

```python
search_organizations(
    query=f"{product} supplier manufacturer distributor wholesale vendor",
    country=[country],
    nace_code=[
        "NACE C: Manufacturing",
        "NACE G: Wholesale and retail trade",
    ],
    keywords_must_any=[
        "supplier", "distributor", "wholesale",
        "manufacturer", "reseller", "vendor", "trading",
    ],
    keywords_must_not=[
        "repair", "rental", "service center", "maintenance", "recruitment",
    ],
    size=12,
    dedup=True,
)
```

## What each lever does

| Lever | Role |
|---|---|
| `nace_code` = **C + G** | The workhorse. Admits makers (C) and traders (G); excludes repair, rental, consulting by category. This is what removes the noise: don't drop it. |
| `query` | Weights the product + supply intent. Self-adjusts: leans distributor for imported goods, manufacturer where made locally. |
| `keywords_must_any` | Forces supply-side language to be present (also required so `must_not` can run). |
| `keywords_must_not` | A soft down-weight, **not** a hard filter: real exclusion comes from NACE. |
| `dedup` | Collapses duplicate company names. |

## Validated example

`product="industrial valves"`, `country="Spain"` → a clean mix of manufacturers and distributors:

| Company | Type | NACE |
|---|---|---|
| Babcock Valves, Tecval, FHT, ICP Valves | Manufacturer | C |
| VAINDUSA, ACOM, DIMAC | Distributor | G |

The same template was validated across MacBooks (UAE, distributor-heavy), hydraulic pumps (Germany, maker-heavy), solar panels (India, mixed), and electronic components (Singapore, trading hub), one parameter set, product-appropriate skew in every geography.

## Notes

<Warning>
  **Don't set `min_score`.** Keywords trigger hybrid retrieval, where a `min_score` of ~0.45 can collapse a clean 12-result query down to 1. Let NACE + keywords do the precision work.
</Warning>

- **Country is activity-based.** GOI resolves location by where a company is *active*, not solely legal HQ: a firm operating in your market can appear even if incorporated abroad. If strict incorporation matters, post-validate.
- **City filters are coarse.** `municipality=["Dubai"]` can return zero (geocoding granularity). Prefer `country`, or `region`/`state` via [`resolve_location`](/mcp/tools).
- **Ignore the `Total` field** in keyword mode: it reports `0` while returning a full page. Count the returned rows.
- **For strict product matching**, add `keywords_must_all=["<core noun>"]` (fewer results, higher precision).
