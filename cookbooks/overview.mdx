---
title: "Cookbooks"
description: "Copy-paste query templates that turn a fuzzy request into a clean, reproducible GOI result."
---

Each cookbook is a **fixed parameter template**: the caller supplies one or two free-text inputs, and everything else is locked in. The quality comes from the parameters, not the search string — so you get the same clean result every time without knowing the internals.

All examples use the [GOI MCP connector](/mcp/index) tool calls (`search_organizations`, `find_similar_organizations`, `aggregate_organizations`). The same parameters map directly to the [Public API](/api/api_start_page) `POST /v2/search`.

<Note>
  **Reproducible within an index version.** Same inputs + same template + same GOI snapshot → same results. Results change as the corpus refreshes; to freeze output, snapshot the returned domains and re-hydrate later via `get_organization_details`.
</Note>
